# Contributing to GetWired

Thanks for your interest in contributing to GetWired! This project is maintained by the [Augment Code community](https://www.reddit.com/r/AugmentCodeAI/) and we welcome contributions from everyone.

## How to Contribute

### 1. Fork the Repository

Click the "Fork" button at the top right of this repo to create your own copy.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR-USERNAME/getwired.git
cd getwired
```

### 3. Create a Branch

Create a new branch for your feature or fix:

```bash
git checkout -b feature/your-feature-name
```

Use a descriptive branch name like `feature/add-new-provider` or `fix/timeout-handling`.

### 4. Make Your Changes

- Install dependencies: `npm install`
- Make your changes
- Test your changes locally

### 5. Commit Your Changes

Write clear, concise commit messages:

```bash
git add .
git commit -m "Add support for new provider X"
```

### 6. Push and Open a Pull Request

```bash
git push origin feature/your-feature-name
```

Then open a Pull Request from your fork to the `main` branch of this repo. Describe what your PR does and link any relevant issues.

## Pull Request Guidelines

- Keep PRs focused on a single change
- Include a clear description of what the PR does and why
- Make sure your code follows the existing style
- Test your changes before submitting
- Be responsive to feedback during code review

## Reporting Bugs

Found a bug? Open an [issue](https://github.com/JaySym-ai/getwired/issues/new) with:

- A clear title and description
- Steps to reproduce the problem
- Expected vs actual behavior
- Your environment (OS, Node version, etc.)

## Suggesting Features

Have an idea? Open an [issue](https://github.com/JaySym-ai/getwired/issues/new) and tag it as a feature request. Describe:

- What you want to see
- Why it would be useful
- Any implementation ideas you have

## Code of Conduct
